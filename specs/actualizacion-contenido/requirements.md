# Requisitos: Actualización de Contenido del Portfolio

## Contexto
El contenido actual del portfolio fue creado cuando Daniel empezó en Ideable (2021). Actualmente lleva **5 años y medio** en la empresa y ha evolucionado significativamente en roles, responsabilidades y tecnologías. Es necesario actualizar toda la información para reflejar su estado actual como profesional.

## Problemas identificados en el contenido actual
1. **Experiencia**: solo menciona tareas iniciales en Ideable (Grails, PostgreSQL, AWS básico). No refleja evolución de rol ni proyectos recientes.
2. **Skills**: desactualizadas. No incluye tecnologías aprendidas en los últimos años.
3. **Proyectos**: solo incluye el portfolio y un toolkit genérico. Faltan proyectos reales, publicaciones, o contribuciones significativas.
4. **About/Bio**: genérica, no transmite el valor actual ni el enfoque actual.
5. **Título/Subtitle**: puede necesitar ajuste si el rol ha evolucionado (ej. Senior, Lead, Researcher).

## Objetivos
- Actualizar experiencia en Ideable con proyectos, logros y evolución de rol real.
- Revisar y actualizar el stack de skills (lenguajes, frameworks, cloud, herramientas).
- Revisar proyectos personales/profesionales destacados.
- Ajustar bio, título y subtítulo para reflejar el perfil actual.
- Mantener consistencia con i18n (español e inglés).

## Criterios de aceptación
- [ ] Experiencia refleja al menos 5.5 años en Ideable con detalle real.
- [ ] Skills incluyen tecnologías actuales y se organizan de forma lógica.
- [ ] Proyectos son representativos y verificables.
- [ ] Bio y título transmiten el valor actual del profesional.
- [ ] Validación con tests E2E tras deploy.

## Notas
- Se realizará mediante entrevista guiada para extraer toda la información actualizada.
- Se documentarán decisiones de contenido en este mismo directorio.
